## 防作弊机制：Commit-Reveal 协议

### 问题
网络延迟可能导致后提交的玩家看到对方选择，造成不公平。

### 解决方案
使用加密承诺机制：

1. **Commit 阶段**（提交承诺）
   - 玩家选择后，生成随机盐值
   - 计算 `hash = SHA256(选择 + 盐值)`
   - 只发送 hash 到服务器，不发送实际选择

2. **Reveal 阶段**（揭示答案）
   - 所有玩家都提交 hash 后
   - 玩家发送实际选择和盐值
   - 服务器验证 `SHA256(选择 + 盐值) == hash`
   - 验证通过才计分

### 代码实现
```javascript
// 客户端生成 hash
async function sha256(message) {
  const msgBuffer = new TextEncoder().encode(message);
  const hashBuffer = await crypto.subtle.digest('SHA-256', msgBuffer);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
}

const mySalt = Math.random().toString(36);
const hash = await sha256(myChoice + mySalt);
socket.emit('commit', hash);

// 服务器验证
const computedHash = crypto.createHash('sha256')
  .update(choice + salt)
  .digest('hex');
if (computedHash !== player.commitment) {
  // 作弊检测
}
```

## 游戏流程
1. 玩家输入昵称加入房间
2. 点击"准备"按钮
3. 所有人准备后，5秒倒计时
4. 倒计时结束前提交选择（发送 hash）
5. 所有人提交后，揭示答案并计分
6. 显示结果（大图标 + 获胜者高亮）

## 关键修复
- **倒计时不显示**: `data.countdow` 拼写错误 → `data.countdown`
- **所有人都出拳头**: `sha256()` 未 await，发送了 Promise 对象 → 添加 `async/await`
- **OAuth 错误**: 缺少 `.env` 文件 → 添加默认配置，OAuth 可选
